Hexo 安装指南

发表于 2016-11-03 | 分类于 “教程”

Hexo官方介绍

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
官网 https://hexo.io/zh-cn/docs/index.html

Hexo博客建立

简洁高效，如同官网的介绍。安装过程非常简单，官网有非常详细的文档，安装文档；并且还是中文的哦！

部署至github

安装完成后需要部署至github，根据官网的介绍，首先进行配置

前期准备:

  • GitHub账号
  • Node.js
  • cnpm
  • Git

第一步

  • 注册好GitHub账号,并安装好Node.js和Git.
  • 右键打开Git Bash,输入
    $ git config --global user.name "你的名字"
    $ git config --global user.email "你的邮箱"
  • 然后到C:\Users\你的用户名下查看是否生成了 .ssh 目录其中是否存在 id_rsa 和id_rsa.pub 文件.
    不存在请在命令行输入 ssh-keygen -t rsa -C "你的邮箱"
    然后一路回车,使用默认值.无需配置密码.
  • 现在打开 .ssh 目录:
    id_rsa 是你的私钥,不要泄露
    id_rsa.pub 则是公钥, 可以放心地告诉任何人.

第二步

  • 登陆 GitHub, 然后如图操作


    填上任意Title，在Key文本框里粘贴 id_rsa.pub 文件的内容:

    然后点击 Add SSH Key,确认添加.
  • 你的个人主页
    点击 Start a project 创建一个库.

    仓库命名格式中的 你的用户名.github.io.

第三步

  • 更换淘宝npm源.
    如果Node.js 安装在C盘,请用使用管理员权限的命令行窗口.
    输入 npm install -g cnpm --registry=https://registry.npm.taobao.org
    等待安装完毕,以后就可以用 cnpm 命令来代替 npm, 除了 publish命令.
  • 安装完淘宝npm源后,输入cnpm install -g hexo-cli 安装Hexo.
  • 在硬盘里寻找一个目录, shift + 右键 选择在此处打开命令窗口.
    输入 hexo init yourname yourname 指的是你的准备存放文件的目录名
    cd youname 进入你指定的目录.
    cnpm install 安装必须文件,等待安装完毕.

    第四步

  • 打开目录下的 _config.yml 文件.
    deploy:
    type: git
    repo: https://github.com/btjoker/btjoker.github.io.git
    ranch: master
    repo: 改成你的提交代码的git地址..
  • 安装 cnpm install hexo-deployer-git --save

第五步

  • Hexo 常用命令
    hexo generate 生成静态页面
    hexo serve 本地预览(默认地址)
    hexo deploy 将.deploy目录部署到GitHub
    hexo new "postName" 新建一个文章
    hexo new page "pageName" 新建页面
    hexo help 查看帮助
    hexo version 查看Hexo的版本
    hexo clean 清理Hexo缓存(更换主题后必须清理)
  • 每次部署的步骤，可按以下三步来进行。
    hexo clean
    hexo generate
    hexo deploy

第六步

  • 主题
    Next 国人写的,有三种模式.非常强大
    Cover A chic theme with facebook-like cover photo
    OishiA white theme based on Landscape plus and Writing.
    TKLA responsive design theme for Hexo. 一个设计优雅的响应式主题
    YiliaResponsive and simple style 优雅简洁响应式主题。
    更多主题请在Hexo官方Theme中寻找 传送门

  • 插件
    hexo-hey非常棒的后台插件,安装简单.
    注意:只能在本地使用.
    更多插件请在Hexo官方Plugins中寻找 传送门

  • 修改主题和作者名字
    打开 你的博客根目录文件 _config.yml
    author: 作者
    theme: 主题

  • 域名绑定
    前提就是有一个域名咯.
    在你的 source 文件夹中创建一个无后缀名大写的 CNAME 文件.
    文件内容为你的 域名,不包括 http://
    如: acjoker.tk

注册一个DNS解析服务网站账号,添加你的域名然后添加纪录

主机记录: @或www
记录类型: CNAME
线路类型: 默认
记录值:你的Github的仓库地址,如 btjoker.github.io
其他的不需要修改.
然后就是等待Dns解析生效.

重要

  • source/_posts 是文章的保存地方
  • _config.yml 站点配置文件
  • 如果更换主题后,请在 .deploy_git 和 public 文件夹存放一个空文件 .nojekyll
    这个是 GitHub 官方给出的更换主题后 CSS/JS 404错误的问题的解决方案.
  • 本教材基于简书的博主潘柏信的补充版.
